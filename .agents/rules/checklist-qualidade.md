Antes de dizer que um RFxx está "pronto", confirme mentalmente:
- [ ] Segue a arquitetura definida na seção 1/2 (stack, pastas)?
- [ ] Os componentes criados são reutilizáveis e isolados (seção 3)?
- [ ] A interface é responsiva em mobile e desktop?
- [ ] Todos os critérios de aceitação do RFxx foram atendidos?
- [ ] O commit segue o padrão `feat: ... (RF0X)`?
- [ ] A entrada correspondente foi registrada no log de prompt engineering?

Se qualquer item falhar, diga isso explicitamente em vez de marcar a tarefa
como concluída.
